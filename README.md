# vision-mcp

A remote MCP server that exposes three vision tools (`ocr_image`, `describe_image`, `answer_image`) backed by Doubao Seed 2.0 Mini on Volcano Ark.

Endpoint: `https://liuxl.com.cn/mcp` (single-user, Bearer-token auth).

See `docs/superpowers/specs/2026-07-03-vision-mcp-design.md` for the full design spec.

## Quick start (local dev)

```bash
git clone https://github.com/Abelliuxl/vision-mcp
cd vision-mcp
uv sync
cp .env.example .env
# fill in ARK_API_KEY in .env
uv run pytest
uv run python -m vision_mcp.server
```

## Deploy

See `deploy/install.sh`. Run on `server94`:

```bash
sudo -E bash deploy/install.sh
```

## Client Configuration

The exposed endpoint is **`https://liuxl.com.cn/mcp`**. Three tools are
exposed: `ocr_image`, `describe_image`, `answer_image`. All require the
`Authorization: Bearer <token>` header; the token is generated by
`deploy/install.sh` and printed at the end (also stored in
`/etc/vision-mcp/env.sh` on server94).

To retrieve the token after deployment:

```bash
ssh server94 'sudo grep VISION_BEARER_TOKEN /etc/vision-mcp/env.sh'
```

### Claude Desktop

Edit `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "vision": {
      "url": "https://liuxl.com.cn/mcp",
      "transport": "streamable-http",
      "headers": {
        "Authorization": "Bearer <paste-token-here>"
      }
    }
  }
}
```

### Cursor

`Settings → MCP → Add new global MCP server`:

- Name: `vision`
- URL: `https://liuxl.com.cn/mcp`
- Headers: `Authorization: Bearer <paste-token-here>`

### Cline

In the Cline panel, open MCP Servers → Configure MCP Servers:

```json
{
  "mcpServers": {
    "vision": {
      "url": "https://liuxl.com.cn/mcp",
      "type": "streamableHttp",
      "headers": {
        "Authorization": "Bearer <paste-token-here>"
      }
    }
  }
}
```

### Cherry Studio / others

Set:

- Base URL: `https://liuxl.com.cn/mcp`
- Custom header: `Authorization: Bearer <paste-token-here>`

## Token Rotation

```bash
ssh server94 'sudo bash -lc "NEW_TOKEN=\$(openssl rand -hex 32) && \
    sed -i \"s|VISION_BEARER_TOKEN=.*|VISION_BEARER_TOKEN=\\\"\$NEW_TOKEN\\\"|\" /etc/vision-mcp/env.sh && \
    sed -i \"s|Bearer [a-f0-9]\\{64\\}|Bearer \$NEW_TOKEN|\" /etc/nginx/sites-enabled/main-page && \
    supervisorctl restart vision-mcp && \
    nginx -t && systemctl reload nginx && \
    echo NEW_TOKEN=\$NEW_TOKEN"'
```

Update every client with the new token.

## Verifying no image is left on disk

```bash
ssh server94 'touch /tmp/vision_mcp_session_start'
# ... make a few tool calls from your client ...
ssh server94 'find /srv/vision-mcp /var/log/vision-mcp /var/lib/nginx/vision_tmp -type f -newer /tmp/vision_mcp_session_start'
```

Expected: empty output (no image-named files; only rolling log files which do not contain image bytes).
